## Ticket: Refactor AprilCam — extract process_frame()

Extract the detection/tracking core from `AprilCam.run()` into a new `process_frame()` method.

### Changes to `src/aprilcam/aprilcam.py`

1. **Add instance attributes** (in `__init__`, after existing initialization):
```python
# Tracking state (initialized by reset_state)
self._prev_gray: Optional[np.ndarray] = None
self._tracks: dict[int, np.ndarray] = {}
self._tag_models: dict[int, AprilTagModel] = {}
self._frame_idx: int = 0
```

2. **Add `reset_state()` method**:
```python
def reset_state(self) -> None:
    self._prev_gray = None
    self._tracks = {}
    self._tag_models = {}
    self._frame_idx = 0
```

3. **Add `process_frame(frame_bgr, timestamp) -> list[TagRecord]`**:
Extract steps 2-5 from `run()` (lines 268-318) into this method:
- Convert to gray
- Detect or LK track based on detect_interval and _frame_idx
- Update playfield
- Filter detections to playfield polygon
- Update/maintain tag models and playfield flows
- Build TagRecord objects from each tag model using AprilTagFlow velocity data
- Increment _frame_idx, update _prev_gray
- Return list of TagRecord

Import `TagRecord` from `aprilcam.detection` at the top.

The velocity data comes from `self.playfield.get_flows()`. For each detected tag, look up its flow and extract vel_px, speed_px. vel_world/speed_world/heading_rad are None for now (will be enhanced later).

4. **Simplify `run()`**:
- Call `self.reset_state()` at the start
- Replace steps 2-5 with `tag_records = self.process_frame(frame, now)`
- Steps 6-7 (speed computation, printing) use the local `vel_ema`/`last_seen` dicts and the tag_models (which are now instance attributes via `self._tag_models`)
- Steps 8-10 remain the same, referencing `self._prev_gray` is not needed since process_frame handles it
- The `frame_idx` local is replaced by `self._frame_idx`

### Important constraints
- `process_frame()` must NOT open windows, call waitKey, or print
- `process_frame()` must NOT read from the camera — it receives the frame
- `run()` behavior must be identical after refactor
- Use `self._frame_idx` instead of local `frame_idx` everywhere in run()
- Use `self._tag_models` instead of local `tag_models` everywhere in run()

### Existing AprilCam.run() code for reference (lines 242-410)
The run() method:
1. Init capture
2. Convert to gray, detect or LK track
3. Update playfield
4. Filter to playfield polygon
5. Update tag models and flows
6. Compute speeds for printing
7. Optional stdout logging
8. Display overlays
9. Keyboard input
10. Bookkeeping (prev_gray, frame_idx)

Steps 2-5 go into process_frame(). Steps 1, 6-10 stay in run().
