## Ticket: Fix velocity noise — port EMA smoothing from run() to process_frame()

### Problem
`process_frame()` in `src/aprilcam/aprilcam.py` uses raw `AprilTagFlow.vel_px` for velocity — a single frame-to-frame delta. On static tags, sub-pixel detector jitter produces fake velocities of 10-100+ px/s.

The original `run()` method had EMA smoothing via `speed_alpha` and `vel_ema`/`last_seen` dicts. This smoothing was left as UI-only locals when `process_frame()` was extracted.

### Fix

1. **Add instance attributes** in `reset_state()`:
```python
def reset_state(self) -> None:
    self._prev_gray = None
    self._tracks = {}
    self._tag_models = {}
    self._frame_idx = 0
    self._vel_ema: dict[int, float] = {}
    self._last_seen: dict[int, tuple[float, float, float]] = {}
```

2. **In `process_frame()`**, after building tag models and before creating TagRecords, compute EMA-smoothed velocity per tag using `self.speed_alpha`. Replace the current velocity lookup:

Current code (around line 338-350):
```python
flow = flows.get(tid)
vel_px_val = flow.vel_px if flow else None
speed_px_val = flow.speed_px if flow else None
tr = TagRecord.from_apriltag(
    model,
    vel_px=vel_px_val,
    speed_px=speed_px_val,
    ...
```

Replace with:
```python
cx, cy = model.center_px
vel_px_val = None
speed_px_val = None
if tid in self._last_seen:
    px, py, pt = self._last_seen[tid]
    dt = max(1e-3, timestamp - pt)
    dx, dy = cx - px, cy - py
    vx, vy = dx / dt, dy / dt
    inst_speed = math.hypot(vx, vy)
    prev_speed = self._vel_ema.get(tid)
    ema_speed = (self.speed_alpha * inst_speed + (1 - self.speed_alpha) * prev_speed) if prev_speed is not None else inst_speed
    self._vel_ema[tid] = ema_speed
    # Dead-band: suppress jitter below 2 px/s
    if ema_speed < 2.0:
        vel_px_val = (0.0, 0.0)
        speed_px_val = 0.0
    else:
        vel_px_val = (vx, vy)
        speed_px_val = ema_speed
self._last_seen[tid] = (cx, cy, timestamp)
tr = TagRecord.from_apriltag(
    model,
    vel_px=vel_px_val,
    speed_px=speed_px_val,
    ...
```

3. **In `run()`**, remove the duplicate `vel_ema` and `last_seen` local dicts and the duplicate speed computation loop (steps 6). Instead, use the TagRecords returned by `process_frame()` which now contain smoothed values. The speed printing in step 7 can iterate over the returned TagRecords.

The run() method currently has local `vel_ema` and `last_seen` dicts (around line 369-370) and a speed computation loop (lines ~388-402). Remove those and update the printing loop to use `tag_record.speed_px` and compute `vang` from `tag_record.vel_px`.

4. **Add `import math`** at top if not already present.
