## Ticket 001: Error handling audit across all MCP tools

Read `src/aprilcam/mcp_server.py` and audit every `@server.tool()` function. Ensure:
- All hardware interactions (cap.read(), registry lookups) are wrapped in try/except
- Every exception path returns `[TextContent(type="text", text=json.dumps({"error": "descriptive message"}))]`
- No unhandled exceptions can reach the MCP framework
- Add a top-level try/except in each tool handler that catches `Exception` and returns a structured error

Pattern to follow:
```python
@server.tool()
async def some_tool(...) -> list[TextContent]:
    try:
        # ... tool logic ...
    except KeyError as e:
        return [TextContent(type="text", text=json.dumps({"error": str(e)}))]
    except Exception as e:
        return [TextContent(type="text", text=json.dumps({"error": f"Unexpected error: {e}"}))]
```

Don't change tools that already have good error handling — just add safety nets where missing.

    Begins continuous AprilTag detection in a background thread. Query
    results with get_tags() or get_tag_history().

        source_id: Camera or playfield ID to detect on.
        family: AprilTag family (default "36h11").

        use_clahe: Apply CLAHE preprocessing.
        use_sharpen: Apply sharpening preprocessing.

        {"error": str} on failure.
    """
