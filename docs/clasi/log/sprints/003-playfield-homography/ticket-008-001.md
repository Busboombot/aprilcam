## Ticket

**Title**: Add create_playfield_from_image MCP tool for static image files

**Description**: Currently, `create_playfield` requires a live camera (`camera_id`). There is no MCP path to create a playfield from a static image file. Add two new MCP tools:

1. `create_playfield_from_image(image_path)` ŌĆö reads a static image file, detects ArUco corners, creates and registers a playfield. Sets camera_id to `"file:<path>"`.
2. `deskew_image(playfield_id, image_path, format)` ŌĆö reads a static image, applies the playfield's deskew, returns the result as base64 ImageContent or file path.

**Acceptance Criteria**:
- [ ] New `create_playfield_from_image(image_path)` MCP tool exists
- [ ] It detects ArUco corners and registers a playfield (camera_id set to "file:<path>")
- [ ] It returns the same JSON shape as `create_playfield` (playfield_id, corners, calibrated)
- [ ] New `deskew_image(playfield_id, image_path, format)` MCP tool exists
- [ ] It reads the image, applies deskew, returns the result as base64 or file
- [ ] Existing `get_playfield_info` and `calibrate_playfield` work with file-based playfields
- [ ] Tests cover both tools using the existing test images
- [ ] All existing tests still pass

## Implementation approach

Modify `src/aprilcam/mcp_server.py` to add two new `@server.tool()` functions.

### `create_playfield_from_image`
- Parameters: `image_path: str`
- Read the image with `cv2.imread(image_path)`, validate it loaded
- Create `Playfield(detect_inverted=True)`, call `pf.update(img)`
- Check polygon detected, report missing corners if not
- Register in `playfield_registry` with `camera_id="file:<image_path>"` and `playfield_id` based on a hash or uuid
- Return same JSON shape as `create_playfield`: `{playfield_id, corners, calibrated}`

### `deskew_image`
- Parameters: `playfield_id: str`, `image_path: str`, `format: str = "base64"`, `quality: int = 85`
- Look up playfield in registry
- Read image with `cv2.imread(image_path)`, validate
- Call `entry.playfield.deskew(img)`
- Encode and return as base64 `ImageContent` or file path (same pattern as `capture_frame`)
