## Ticket

**Title**: Mask source image to playfield polygon before deskew warp

**Description**: `Playfield.deskew()` uses `cv2.warpPerspective` on the full source image, so pixels outside the playfield polygon bleed into the output. The output boundary should be exactly at the ArUco corner marker centers — nothing outside should be visible.

**Acceptance Criteria**:
- [ ] `Playfield.deskew()` masks source image to playfield polygon before warping
- [ ] Output contains only content from inside the playfield boundary
- [ ] Pixels outside the polygon are black (0,0,0) in the output
- [ ] All existing tests still pass

## File to modify

`src/aprilcam/playfield.py` — the `deskew` method (lines 122-135).

Current code:
```python
def deskew(self, frame_bgr: np.ndarray) -> np.ndarray:
    if self._poly is None:
        return frame_bgr
    UL, UR, LR, LL = self._poly.astype(np.float32)
    w_top = float(np.linalg.norm(UR - UL))
    w_bottom = float(np.linalg.norm(LR - LL))
    h_left = float(np.linalg.norm(LL - UL))
    h_right = float(np.linalg.norm(LR - UR))
    out_w = max(10, int(round(max(w_top, w_bottom))))
    out_h = max(10, int(round(max(h_left, h_right))))
    src = np.array([UL, UR, LR, LL], dtype=np.float32)
    dst = np.array([[0, 0], [out_w - 1, 0], [out_w - 1, out_h - 1], [0, out_h - 1]], dtype=np.float32)
    M = cv.getPerspectiveTransform(src, dst)
    return cv.warpPerspective(frame_bgr, M, (out_w, out_h))
```

## Implementation

Before the `warpPerspective` call, create a mask from the polygon and apply it to the source image:

```python
# Mask source to playfield polygon
mask = np.zeros(frame_bgr.shape[:2], dtype=np.uint8)
cv.fillConvexPoly(mask, self._poly.astype(np.int32), 255)
masked = cv.bitwise_and(frame_bgr, frame_bgr, mask=mask)
return cv.warpPerspective(masked, M, (out_w, out_h))
```
